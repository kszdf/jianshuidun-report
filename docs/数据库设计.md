# 数据库设计文档

## 设计原则
- 单租户起步，预留公司ID字段，未来做多租户改造方便
- 凭证+余额双数据源，既支持明细查询也支持快速汇总
- 辅助核算用JSON字段存，灵活适配不同财务软件的核算维度

## 报表数据计算方式

### 资金日报
- 来源：`acc_subject_balance`（库存现金+银行存款+其他货币资金 期末余额）
- 当日收支：`acc_voucher_item` 按日期汇总
- 多账户：`acc_cashflow` 按账户汇总

### 项目利润表
- 收入：凭证明细中收入类科目，按项目辅助核算汇总
- 成本：凭证明细中成本类科目，按项目辅助核算汇总
- 毛利 = 收入 - 成本
- 利润率 = 毛利 / 收入

### 应收账龄表
- 应收余额：应收账款科目余额 - 预收账款科目余额
- 账龄计算：按最后一笔收款日期到今天的天数
- 来源：凭证明细中的应收/预收科目，按客户辅助核算分组

## 数据质量评分维度

1. 科目映射覆盖率（30分）：已映射科目 / 全部科目
2. 辅助核算完整率（30分）：项目/客户/供应商核算完整的凭证占比
3. 凭证借贷平衡（20分）：平衡的凭证数 / 总凭证数
4. 期间完整性（20分）：有数据的月份 / 应有数据的月份
