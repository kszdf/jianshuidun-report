## 一、准备工作

### 1.1 服务器信息记录
先记下你的服务器信息，后面要用到：

| 项目 | 你的信息 | 示例 |
|------|---------|------|
| 服务器公网IP | 自己填 | 123.45.67.89 |
| 操作系统 | 自己看 | Ubuntu 22.04 / CentOS 7.9 |
| root密码 | 自己记 | ******** |
| 服务器所在地 | 自己看 | 上海 / 广州 |

> 在哪看？登录腾讯云控制台 → 云服务器 → 实例列表，就能看到。

### 1.2 域名解析（必须做，不然打不开）

1. 登录腾讯云 → 域名注册 → 找到 hgtcs.com → 管理
2. 左侧「DNS解析」→ 「添加记录」
3. 添加两条记录：

**记录一（推荐：子域名访问）：**
- 主机记录：`report`（或 `jianshuidun`，你喜欢什么就填什么）
- 记录类型：`A`
- 记录值：`你的服务器公网IP`
- TTL：默认 600 就行

**记录二（如果想用主域名访问）：**
- 主机记录：`@`
- 记录类型：`A`
- 记录值：`你的服务器公网IP`

> 💡 建议先用子域名 report.hgtcs.com，主域名留给官网。
> ⚠️ 域名必须已经备案！没备案的域名解析到国内服务器会被拦截。

### 1.3 安全组放行端口（必须做）

1. 腾讯云控制台 → 云服务器 → 你的实例 → 安全组
2. 点击安全组名称 → 「入站规则」→「添加规则」
3. 添加以下两条：

| 类型 | 来源 | 端口 | 说明 |
|------|------|------|------|
| HTTP | 0.0.0.0/0 | 80 | 网页访问必需 |
| HTTPS | 0.0.0.0/0 | 443 | 加密访问必需 |

> 22端口（SSH登录）默认应该已经开了，没开的话也加上。

## 二、登录服务器

### 方式一：腾讯云网页登录（最简单，推荐新手用）

1. 腾讯云控制台 → 云服务器 → 实例列表 → 找到你的服务器
2. 点击「登录」→「标准登录方式」→ 输入root密码 → 登录
3. 进入一个黑色命令行窗口就成功了

### 方式二：本地SSH登录（Windows）

1. 按 Win+R，输入 `cmd`，回车
2. 输入：`ssh root@你的服务器IP`
3. 输入密码（输入时不显示字符，正常现象）

### 方式三：本地SSH登录（Mac）

1. 打开「终端」
2. 输入：`ssh root@你的服务器IP`
3. 输入密码

## 三、安装运行环境

> ⚠️ 下面所有命令都要一行一行复制粘贴到服务器命令行里，回车执行。
> 看到 `[sudo] password` 就输入密码；看到 `Do you want to continue?` 就输入 `y` 回车。

### 3.1 先判断你的系统是 Ubuntu 还是 CentOS

输入以下命令：
```bash
cat /etc/os-release
```

- 看到 `Ubuntu` 字样 → 走「3.2 Ubuntu 系统」
- 看到 `CentOS` 字样 → 走「3.3 CentOS 系统」

### 3.2 Ubuntu 系统（推荐，文档以Ubuntu为例）

```bash
# 第一步：更新软件源
apt update -y

# 第二步：安装Python、Git、Nginx等基础工具

# 第三步：验证安装是否成功
python3 --version    # 应该显示 Python 3.x.x
nginx -v             # 应该显示 nginx version
git --version        # 应该显示 git version
```

三个都正常显示就继续。

### 3.3 CentOS 系统

```bash
# 第一步：安装基础工具
yum install -y python3 python3-pip python3-devel nginx git gcc

# 第二步：验证
python3 --version
nginx -v
git --version
```

### 4.1 创建项目目录并下载代码

```bash
# 进入 /opt 目录（Linux下放应用软件的标准位置）
cd /opt
```

> 激活成功后，命令行前面会出现 `(venv)` 字样。

## 五、配置后台服务（开机自启）

现在每次启动都得手动输命令太麻烦，我们配置成系统服务，开机自动跑，挂了自动重启。

### 5.1 创建服务文件

```bash
# 用 nano 编辑器创建服务文件
nano /etc/systemd/system/jianshuidun.service
```

然后把下面这段**完整复制**进去（注意路径别错）：

```ini
[Unit]
Description=JianShuiDun Report Service
After=network.target

[Service]
User=root
WorkingDirectory=/opt/jianshuidun-report/backend
Environment="PATH=/opt/jianshuidun-report/backend/venv/bin"
ExecStart=/opt/jianshuidun-report/backend/venv/bin/gunicorn main:app -w 2 -k uvicorn.workers.UvicornWorker --bind 127.0.0.1:8000 --timeout 120
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

**怎么粘贴保存退出：**
- 右键粘贴（网页终端），或者 Ctrl+Shift+V
- 按 `Ctrl+O` 保存（字母O，不是数字0）
- 按回车确认文件名
- 按 `Ctrl+X` 退出

### 5.2 启动服务

```bash
# 重新加载服务配置
systemctl daemon-reload

# 启动服务
systemctl start jianshuidun

# 设置开机自启
systemctl enable jianshuidun

# 查看状态
systemctl status jianshuidun
```

看到 `active (running)` 绿色字样，说明服务已经在后台跑起来了。

按 `q` 退出状态查看。

## 六、配置Nginx反向代理

Nginx 负责：
- 用户访问 80 端口（网页默认端口）
- 静态文件（前端页面）直接由 Nginx 发送，速度快
- API 请求转发给后端 8000 端口

### 6.1 Ubuntu 配置方式

```bash
# 创建站点配置文件
nano /etc/nginx/sites-available/jianshuidun
```

复制下面的内容进去，**把 report.hgtcs.com 改成你自己的域名**：

```nginx
server {
    listen 80;
    server_name report.hgtcs.com;

    # 前端静态文件
    location / {
        root /opt/jianshuidun-report/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }

    # API文档（可选，正式上线可以注释掉）
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
        proxy_set_header Host $host;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
        proxy_set_header Host $host;
    }
}
```

保存退出后，启用站点：

```bash
# 启用站点
ln -s /etc/nginx/sites-available/jianshuidun /etc/nginx/sites-enabled/

# 禁用默认站点（可选，避免冲突）
unlink /etc/nginx/sites-enabled/default 2>/dev/null; echo "done"

# 测试配置是否正确
nginx -t
```

看到 `test is successful` 就没问题。

```bash
# 重载Nginx
systemctl reload nginx
```

### 6.2 CentOS 配置方式

```bash
nano /etc/nginx/conf.d/jianshuidun.conf
```

内容和上面一模一样，保存退出。

```bash
nginx -t
systemctl reload nginx
```

浏览器打开：`http://report.hgtcs.com`（换成你的域名）

能看到登录页面就成功了！

## 七、配置HTTPS（小绿锁）

> 💡 强烈建议配置。没有HTTPS，浏览器地址栏会显示「不安全」，影响用户信任感。

### 7.1 安装证书申请工具

**Ubuntu：**
```bash
apt install -y certbot python3-certbot-nginx
```

**CentOS：**
```bash
yum install -y certbot python3-certbot-nginx
```

### 7.2 自动申请并配置证书

```bash
certbot --nginx -d report.hgtcs.com
```

> 把 report.hgtcs.com 换成你的域名。

执行过程中会问你几个问题：
1. 输入邮箱 → 输你的邮箱
2. 同意服务条款 → 输入 `Y`
3. 是否接收邮件 → 输入 `N`（不需要）
4. 是否自动跳转HTTPS → 选 `2`（Redirect，推荐，HTTP自动跳HTTPS）

### 7.3 验证

浏览器打开：`https://report.hgtcs.com`

地址栏有小锁头图标，就说明HTTPS配置成功了。

### 7.4 关于证书续期

Certbot 会自动配置定时任务，证书快到期前自动续期，不用你管。

可以手动测试一下续期能不能正常工作：
```bash
certbot renew --dry-run
```

看到 `Congratulations, all simulated renewals succeeded` 就没问题。

### 9.6 备份数据

数据库文件就在这里：`/opt/jianshuidun-report/backend/data/jianzao.db`

定期复制一份就行：
```bash
# 备份
cp /opt/jianshuidun-report/backend/data/jianzao.db /opt/jianzao_backup_$(date +%Y%m%d).db

# 查看备份列表
ls -la /opt/jianzao_backup_*.db
```

建议设置定时自动备份：
```bash
# 每天凌晨2点自动备份
(crontab -l 2>/dev/null; echo "0 2 * * * cp /opt/jianshuidun-report/backend/data/jianzao.db /opt/jianzao_backup_\$(date +\%Y\%m\%d).db") | crontab -

# 只保留最近30天的备份（可选）
(crontab -l 2>/dev/null; echo "0 3 * * * find /opt -name 'jianzao_backup_*.db' -mtime +30 -delete") | crontab -
```

### 9.7 修改默认密码（⚠️ 重要！上线前必改）

```bash
# 进入后端目录
cd /opt/jianshuidun-report/backend
source venv/bin/activate

# 启动Python交互环境
python3
```

然后在Python里执行（把新密码换成你想要的）：

```python
from app.core.database import SessionLocal
from app.models.system import User
from app.core.security import get_password_hash

db = SessionLocal()

# 修改boss密码
boss = db.query(User).filter(User.username == "boss").first()
if boss:
    boss.hashed_password = get_password_hash("你的新密码")
    db.commit()
    print("boss密码已修改")

# 修改finance密码
finance = db.query(User).filter(User.username == "finance").first()
if finance:
    finance.hashed_password = get_password_hash("你的新密码")
    db.commit()
    print("finance密码已修改")

db.close()
exit()
```

> ⚠️ 默认密码一定要改！不然谁都能登录你的系统。

## 十、常见问题排查

### Q1: 浏览器访问显示「无法访问此网站」

**排查步骤：**
1. 先试 IP 能不能访问：`http://你的服务器IP:8000/docs`
   - IP也打不开 → 安全组没放行端口，回去看 1.3 节
   - IP能打开，域名打不开 → DNS解析没生效，等一会儿或检查解析记录

2. 检查Nginx状态
```bash
systemctl status nginx
```

### Q2: 页面打开是404或者Nginx默认页

检查配置文件有没有放对位置：
```bash
# Ubuntu
ls -la /etc/nginx/sites-enabled/

# CentOS
ls -la /etc/nginx/conf.d/
```

确认 jianshuidun 配置文件存在，然后重载：
```bash
nginx -t
systemctl reload nginx
```

### Q3: 后端服务起不来

看错误日志：
```bash
journalctl -u jianshuidun -n 50 --no-pager
```

把报错信息截图发给我，我帮你看。

### Q4: 登录后接口报错 500

多半是数据库初始化有问题，重新初始化一下：
```bash
cd /opt/jianshuidun-report/backend
source venv/bin/activate
python init_data.py
systemctl restart jianshuidun
```

### Q5: HTTPS证书申请失败

检查三件事：
1. 域名解析是否生效（ping一下你的域名，看IP对不对）
2. 80端口有没有开放（安全组和防火墙都要开）
3. 域名有没有备案（国内服务器必须备案）
