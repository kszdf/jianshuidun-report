## 本地开发环境搭建

### 前置要求
- Python 3.10+
- Node.js 16+（前端工程化版本需要）
- Git

### 一、后端启动

```bash
# 1. 克隆代码
git clone https://github.com/kszdf/jianshuidun-report.git
cd jianshuidun-report/backend

# 2. 创建虚拟环境
python -m venv venv
# Windows: venv\Scripts\activate
# Mac/Linux: source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 初始化数据（首次运行）
python init_data.py

# 5. 启动服务
uvicorn main:app --reload --port 8000
```

启动后访问：
- API文档：http://localhost:8000/docs
- 前端页面：http://localhost:8000/（静态文件由后端直接提供）

### 二、前端（CDN单文件版）

无需构建，直接打开 `frontend/index.html` 即可。

默认连接本地后端API地址为 `http://localhost:8000/api`，如需修改，打开 index.html 搜索 `API_BASE` 变量。

### 三、前端（工程化版本）

```bash
cd frontend-vite
npm install
npm run dev
```

访问 http://localhost:5173

## 生产环境部署

### 推荐配置
- 服务器：2核4G（最低1核2G也能跑）
- 系统：Ubuntu 22.04 / CentOS 7+
- 数据库：SQLite（起步用）→ PostgreSQL（数据量大了再换）
- Web服务器：Nginx（反向代理+静态文件）
- 进程管理：systemd / Supervisor

### 架构图

```
用户浏览器 → Nginx(80/443) → 静态文件(前端)
                    ↓
             FastAPI(8000) → SQLite/PostgreSQL
```

## 云服务器部署步骤详解

### 第一步：登录服务器

```bash
ssh root@你的服务器IP
```

### 第二步：安装基础环境

```bash
# Ubuntu/Debian
apt update
apt install -y python3 python3-pip python3-venv nginx git

# CentOS
yum install -y python3 python3-pip nginx git
```

### 第三步：拉取代码

```bash
cd /opt
git clone https://github.com/kszdf/jianshuidun-report.git
cd jianshuidun-report
```

### 第四步：后端部署

```bash
cd backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
pip install uvicorn gunicorn

# 初始化数据
python init_data.py
```

### 第五步：配置 systemd 服务

```bash
vim /etc/systemd/system/jianshuidun.service
```

写入以下内容：

```ini
[Unit]
Description=JianShuiDun Report Service
After=network.target

[Service]
User=root
WorkingDirectory=/opt/jianshuidun-report/backend
Environment="PATH=/opt/jianshuidun-report/backend/venv/bin"
ExecStart=/opt/jianshuidun-report/backend/venv/bin/gunicorn main:app -w 2 -k uvicorn.workers.UvicornWorker --bind 127.0.0.1:8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
systemctl daemon-reload
systemctl start jianshuidun
systemctl enable jianshuidun
systemctl status jianshuidun
```

### 第六步：配置 Nginx

```bash
vim /etc/nginx/sites-available/jianshuidun
```

写入：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    # 前端静态文件
    location / {
        root /opt/jianshuidun-report/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # API文档
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
        proxy_set_header Host $host;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
        proxy_set_header Host $host;
    }
}
```

启用配置：

```bash
# Ubuntu
ln -s /etc/nginx/sites-available/jianshuidun /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx

# CentOS
# 把配置放到 /etc/nginx/conf.d/jianshuidun.conf
nginx -t
systemctl reload nginx
```

### 第七步：开放端口

腾讯云/阿里云控制台 → 安全组 → 放行 80 端口（HTTP）、443 端口（HTTPS）。

### 第八步：配置 HTTPS（可选但推荐）

```bash
# 安装 certbot
apt install -y certbot python3-certbot-nginx

# 自动申请证书
certbot --nginx -d 你的域名
```

证书会自动续期，不用管。

## 常见问题

### Q: 后端启动报错？
先看报错信息，90%的问题是依赖没装全。执行 `pip install -r requirements.txt` 再试。

### Q: 前端打开空白？
F12打开控制台看报错。大概率是API地址不对。CDN版改 index.html 里的 `API_BASE`。

### Q: SQLite 数据量大了卡怎么办？
换 PostgreSQL。改 `backend/app/core/database.py` 里的数据库连接串，数据用 `postgresql://用户:密码@localhost/数据库名`。

### Q: 怎么更新代码？
```bash
cd /opt/jianshuidun-report
git pull
systemctl restart jianshuidun
```

### Q: 数据存在哪里？
SQLite 数据库文件在 `backend/data/jianzao.db`，定期备份这个文件就行。

```bash
# 备份
cp /opt/jianshuidun-report/backend/data/jianzao.db /backup/jianzao_$(date +%Y%m%d).db
```
